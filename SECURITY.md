# Security Policy

Report vulnerabilities privately to the maintainers before opening a public issue. Do not include credentials, retailer session tokens, receipts, local databases, diagnostics bundles, LAN pairing payloads, or screenshots containing personal data in public GitHub issues.

Sensitive desktop data includes:

- retailer login/session artifacts
- receipt databases, exports, document storage, and OCR source files
- local backup bundles
- LAN mobile pairing secrets and temporary bridge endpoints
- GlitchTip/Sentry DSNs, auth tokens, source-map tokens, VPS credentials, and signing credentials

Public issues are appropriate for reproducible bugs after removing private data. Security reports should include the affected version, platform, impact, and sanitized reproduction steps.
